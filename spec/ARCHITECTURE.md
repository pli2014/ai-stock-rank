# AI股票趋势分析系统架构文档

## 1. 架构概述

本系统采用单体Web应用架构，基于Flask框架构建，专注于A股股票的技术面分析。系统通过两阶段分析策略实现高效的数据处理：

- **第一阶段**：限流下载股票日线数据到本地缓存（避免API限流）
- **第二阶段**：并发分析已缓存的股票数据（32线程并行处理）

## 2. 核心组件

### 2.1 数据获取层 (`stock_data.py`)
- **功能**：负责从akshare获取股票基础数据
- **缓存策略**：股票日线数据缓存1天，确保数据新鲜度
- **接口**：
  - `get_stock_list()`: 获取满足市值条件的股票列表
  - `get_stock_data()`: 获取单只股票的日线数据

### 2.2 分析引擎层 (`analyzer.py`)
- **功能**：实现股票趋势分析算法
- **核心类**：`StockTrend` - 封装股票分析结果
- **分析维度**：
  - 30日涨幅 (>10%)
  - 平均换手率 (<20%)
  - 上涨交易日数 (>10/15/20/25天之一)
  - 逐步拉升趋势验证
  - 最近5日连续上涨验证

### 2.3 Web服务层 (`app.py`)
- **功能**：提供HTTP接口和Web界面
- **路由**：
  - `/`: 主页
  - `/analyze`: 分析报告页面
  - `/api/analysis`: JSON API接口
  - `/api/status`: 分析进度查询
- **缓存策略**：分析结果缓存24小时（当天有效）

### 2.4 前端展示层 (`templates/`)
- **模板**：基于Jinja2的HTML模板
- **响应式设计**：支持桌面和移动设备
- **交互功能**：支持切换显示所有股票或仅符合条件股票

## 3. 数据流

```
用户请求 → Flask路由 → perform_analysis()
    ↓
检查缓存 → 有缓存？ → 返回缓存结果
    ↓ 否
获取股票列表(min_market_cap≥100亿) → 下载股票数据(限流) → 并发分析 → 生成报告 → 缓存结果
```

## 4. 性能优化

### 4.1 两阶段并发策略
- **下载阶段**：8线程限流下载，避免akshare API限流
- **分析阶段**：32线程并发分析，充分利用CPU资源

### 4.2 多级缓存
- **L1缓存**：股票日线数据（1天有效期）
- **L2缓存**：完整分析结果（24小时有效期）

### 4.3 内存优化
- 使用pandas高效处理数据
- 及时释放不必要的DataFrame引用

## 5. 扩展性设计

### 5.1 模块化架构
各组件职责单一，便于单独测试和替换：
- 数据获取模块可替换为其他数据源
- 分析引擎可扩展新的技术指标
- Web层可独立部署或集成到其他系统

### 5.2 配置驱动
关键参数通过URL参数配置：
- `min_market_cap`: 最小市值筛选条件
- `limit`: 分析股票数量上限
- `max_workers`: 并发线程数
- `refresh`: 强制刷新标志

## 6. 安全考虑

- 输入参数类型验证和范围检查
- 异常处理和错误页面
- 并发控制避免重复分析任务
- JSON序列化处理numpy类型安全

## 7. 部署方案

### 7.1 开发环境
- 直接运行 `python app.py`
- 使用 `start_web.bat` / `start_web.sh` 启动脚本

### 7.2 生产环境建议
- 使用Gunicorn + Nginx部署
- 配置适当的worker进程数
- 设置日志轮转和监控
- 定期清理cache目录

## 8. 测试策略

- 单元测试：`test_criteria.py` 验证选股条件逻辑
- 集成测试：`test_json_fix.py` 验证数据序列化
- 手动测试：通过Web界面验证端到端功能
