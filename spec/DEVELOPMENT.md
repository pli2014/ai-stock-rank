# 开发规范

## 1. 环境准备

### 1.1 Python版本
- 推荐使用 Python 3.8+

### 1.2 依赖安装
```bash
pip install -r requirements.txt
```

### 1.3 必需的依赖包
- `flask`: Web框架
- `akshare`: 股票数据获取
- `pandas`: 数据处理
- `numpy`: 数值计算

## 2. 代码规范

### 2.1 命名规范
- 变量名：snake_case（小写下划线）
- 函数名：snake_case
- 类名：PascalCase
- 常量：UPPER_SNAKE_CASE

### 2.2 代码风格
- 遵循 PEP 8 规范
- 行长度不超过 88 字符
- 使用类型注解
- 函数和类要有文档字符串

### 2.3 注释规范
- 复杂逻辑必须添加注释
- 注释使用中文
- 注释要解释"为什么"而不是"是什么"

### 2.4 错误处理
- 所有外部API调用必须有异常处理
- 返回用户友好的错误信息
- 记录详细的错误日志（开发环境）

## 4. 开发流程

### 4.1 功能开发
1. 在本地创建特性分支
2. 实现功能并添加相应测试
3. 运行现有测试确保不破坏现有功能
4. 更新相关文档（如需要）
5. 提交代码并创建合并请求

### 4.2 Bug修复
1. 复现问题并编写测试用例
2. 修复问题并确保测试通过
3. 验证修复不会引入新问题
4. 更新文档（如需要）

### 4.3 性能优化
- 优先优化算法复杂度
- 其次考虑并发和缓存策略
- 最后考虑底层库替换
- 每次优化都要有性能测试对比

## 5. 测试规范

### 5.1 单元测试
- 测试边界条件和异常情况
- 覆盖核心业务逻辑
- 使用mock避免外部依赖

### 5.2 集成测试
- 测试端到端的数据流
- 验证缓存机制正确性
- 测试并发场景

### 5.3 手动测试
- Web界面功能验证
- 不同参数组合测试
- 边界值和异常输入测试

## 6. 提交规范

### 6.1 提交信息格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

### 6.2 Type类型
- `feat`: 新功能
- `fix`: Bug修复  
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具变动

### 6.3 示例
```
feat(analyzer): 添加市值和成交额字段支持

- 扩展StockTrend类包含market_cap和avg_volume
- 修改build_stock_trend函数提取相关数据
- 更新报告模板显示新增字段

Closes #123
```

## 7. 部署规范

### 7.1 开发环境部署
```bash
# Linux/Mac
chmod +x start_web.sh
./start_web.sh

# Windows  
start_web.bat
```

### 7.2 生产环境部署
- 使用虚拟环境隔离依赖
- 配置适当的并发参数
- 设置监控和日志收集
- 定期备份和清理缓存

## 8. 性能监控

### 8.1 关键指标
- 分析任务完成时间
- 内存使用情况
- API响应时间
- 缓存命中率

### 8.2 监控方法
- Flask内置的性能分析
- 系统资源监控工具
- 自定义日志记录关键路径耗时

## 9. 安全规范

### 9.1 输入验证
- 所有用户输入必须验证类型和范围
- 防止SQL注入和XSS攻击
- 限制文件上传和路径遍历

### 9.2 依赖安全
- 定期更新依赖包
- 检查已知安全漏洞
- 使用最小权限原则

### 9.3 数据安全
- 敏感数据不要记录在日志中
- 缓存文件设置适当权限
- 定期清理临时数据
